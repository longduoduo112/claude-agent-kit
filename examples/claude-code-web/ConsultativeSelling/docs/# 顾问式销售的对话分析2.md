## 二、逐轮深度拆解

### 阶段1：资格确认（建立对话节奏）

```
AI: "行，那就不兜圈子了。"
AI: "找到我们，应该对AI员工或者说AI私域有点想法。"
AI: "你是做什么业务的，提供什么产品或服务？"
```

**表面动作：** 询问用户业务

**隐藏目标：**
| 维度 | 内容 |
|------|------|
| 筛选意图 | 确认对方是否是目标客户（有私域、有销售场景） |
| 节奏控制 | "不兜圈子"设定直接、高效的对话基调 |
| 心理暗示 | "找到我们，应该有想法"——暗示对方是主动来的，有需求 |
| 信息收集 | 为后续"展示行业认知"做准备 |

**底层原理：** 顾问式销售的第一步不是推销，是**资格确认**——确认对方值得花时间深聊。

### 阶段2：痛点挖掘（引导用户自己说出痛点）

```
用户: "护肤和医美"

AI: "哦，医美护肤，这行我熟。"
AI: "客户问题是不是又多又杂，还特别喜欢半夜三更来咨询？"
AI: "那你现在主要的痛点是啥？"

AI: "还是觉得顾问水平参差不齐，话术不标准，导致转化率上不去？"
```

**表面动作：** 询问痛点

**隐藏目标与技巧拆解：**

| 句子 | 技巧 | 作用 |
|------|------|------|
| "这行我熟" | 行业认知展示 | 建立"我懂你"的信任感 |
| "客户问题是不是又多又杂，半夜三更来咨询" | **嘴替**：替用户描述场景 | 让用户觉得"他懂我的日常" |
| "是A，还是B？" | **二选一提问** | 不让用户思考"有没有痛点"，而是"哪个痛点更痛" |

**底层原理：**

```yaml
传统提问: "你有什么痛点？"（开放式，用户要思考）
高手提问: "是A，还是B？"（封闭式，用户只需选择）

二选一的设计原则:
  - A和B都是你能解决的问题
  - A和B覆盖了目标客户的主要痛点
  - 用户无论选哪个，都进入你的节奏
```

**隐藏目标与技巧拆解：**

| 句子 | 技巧 | 作用 |
|------|------|------|
| "这个是医美行业通病了" | **正常化** | 让用户觉得"我的问题不丢人"，降低防御 |
| "专业度不够→客户不信任→高客单推不出去" | **因果链条** | 帮用户看到问题的后果 |
| "你们肯定也搞过培训/SOP吧？" | **预判用户试过的方案** | 展示你懂行业，为"堵路"铺垫 |
| "为啥没解决？是A还是B？" | **追问替代方案失败的原因** | 让用户自己说出"为什么其他方法没用" |

**底层原理：SPIN的内化**

```yaml
这里运用的是SPIN的P（Problem）和I（Implication）:

P_痛点: 话术不标准，专业知识不够
  ↓ (导致)
I_后果: 客户不信任 → 高客单推不出去

同时预判用户的替代方案:
  - 内部培训
  - 话术SOP
  
追问为什么没用:
  - 员工不执行？
  - 流失太快，培训跟不上？
  
目的: 让用户自己说出"培训这条路走不通"的原因
```

**这是整段对话最关键的一轮。** 让我逐句拆解：

| 句子 | 技巧 | 深层作用 |
|------|------|----------|
| "你这个问题太典型了" | **命名+正常化** | 让用户觉得"你见过很多这样的情况"，建立权威 |
| "你最专业的顾问根本没机会出手" | **问题重新描述** | 用更精准的语言复述用户的问题 |

**底层原理：问题重新定义**

```yaml

  "前端引流的钱，大部分都打了水漂"

重新定义的价值:
  1. 从"人的问题"变成"流程/结构问题"
  2. 从"能力不足"变成"资源错配"
  3. 从"模糊的感觉"变成"可量化的损失"
```

```
AI: "至于资料不全，这太正常了，十个老板九个都这么说。"
AI: "关键不在于资料有多'全'，而在于我们有没有一个'标准'。"
AI: "你那个金牌顾问，就是最好的'活教材'和'标准答案'。"
AI: "只要她还在，就算你一份资料都没有，我们也能通过访谈、复盘她的聊天记录，
     把她的知识体系和话术技巧给'扒'下来，做成AI的内核。"
```

**这是非常精彩的顾虑处理，拆解一下：**

| 句子 | 技巧 | 作用 |
|------|------|------|
| "有什么给什么，别有压力" | **降低门槛** | 消除"资料不够"的焦虑 |
| "十个老板九个都这么说" | **正常化** | 让用户知道这是普遍情况 |
| "关键不在于'全'，而在于有没有'标准'" | **重新定义问题** | 把"资料不全"转化为"有没有标杆" |
| "金牌顾问就是最好的活教材" | **提供替代方案** | 即使没资料也能解决 |
| "我们能把她的知识体系'扒'下来" | **展示专业能力** | 让用户知道你们有办法 |

**底层原理：重新定义问题**

```yaml
用户担心: "我资料不全"

错误回应: "没关系，我们可以帮你补全"（承认"全"很重要）

正确回应: "关键不在于'全'，而在于有没有'标准'"（重新定义什么是重要的）

然后给出替代方案: "金牌顾问本人就是最好的资料"
```

---

**预期管理：**

```
AI: "我们做这个是专业的，先帮你从0到1搭个框架，快速跑起来解决80%的问题。"
AI: "后面再慢慢优化，迭代到95分。"
```

**为什么要说"80%"和"95分"而不是"100%"？**

```yaml
预期管理的作用:
  1. 坦诚: 不过度承诺，建立信任
  2. 降压: "先解决80%"比"完美解决"更容易启动
  3. 铺垫: 为后续持续服务留空间
```

---

**推进下一步：**

```
AI: "你那个金牌顾问，配合度怎么样？只要她愿意配合，这事就好办。"
```

**这句话的精妙之处：**

```yaml
表面上: 询问金牌顾问的配合度
实际上: 
  1. 假设用户已经决定做了（"只要她愿意配合，这事就好办"）
  2. 把决策焦点从"要不要做"转移到"金牌顾问配不配合"
  3. 推进到下一个具体环节
```

## 三、提炼底层方法论

### 3.1 客户旅程与阶段隐藏目标

```yaml
顾问式销售的五阶段隐藏目标:

  阶段1_资格确认:
    表面动作: 询问用户业务
    隐藏目标: 筛选是否是目标客户
    核心技巧: 设定直接高效的对话基调
    
  阶段2_痛点挖掘:
    表面动作: 询问用户痛点
    隐藏目标: 让用户自己说出痛点（而不是你告诉他）
    核心技巧: 
      - 行业认知展示（"这行我熟"）
      - 二选一提问（不问有没有，问哪个更痛）
      - 预判替代方案（"你们肯定也搞过..."）
    
  阶段3_问题诊断:
    表面动作: 分析问题
    隐藏目标: 重新定义问题，建立专业权威
    核心技巧:
      - 问题重新描述（用更精准的语言）
      - 帮用户算账（把模糊感觉变成具体损失）
      - 堵住替代方案（解释为什么其他方法不行）
    
  阶段4_方案呈现:
    表面动作: 介绍解决方案
    隐藏目标: 用新概念锚定方案价值
    核心技巧:
      - 概念重塑（"AI客服"→"克隆金牌顾问"）
      - 步骤清晰化（三步走）
      - 价值具象化（复制无数份，安插在所有入口）
    
  阶段5_落地推进:
    表面动作: 说明具体操作
    隐藏目标: 降低门槛，消除顾虑，推进下一步
    核心技巧:
      - 重新定义用户角色（你只需要提供原料）
      - 主动消除技术/安全/资料顾虑
      - 预期管理（先80%，再95%）
      - 焦点转移（从"要不要做"到"具体怎么配合"）
```

### 3.2 核心思维模型

#### 模型1：二选一框架控制

```yaml
原理: 永远不问开放式问题，用二选一引导用户进入你的框架

应用场景:
  痛点挖掘: "是A问题，还是B问题？"
  原因追问: "是因为A，还是因为B？"
  方案确认: "你想先了解A，还是直接看B？"
  推进下一步: "你看是这周方便，还是下周？"

设计原则:
  - A和B都是你能解决/你希望的方向
  - A和B覆盖了主要可能性
  - 用户选哪个都在你的节奏里
```

#### 模型2：问题重新定义

```yaml
原理: 用户描述的是表象，你要帮他看到本质

操作步骤:
  1. 听用户的原始描述
  2. 用更精准的语言重新描述（展示你理解了）
  3. 揭示背后的本质问题
  4. 把模糊感觉变成可量化的损失

示例:
  用户描述: "美容师和老板娘不专业，她们接触的客户最多"
  重新描述: "你最专业的顾问没机会出手，客户在第一关就被挡回去了"
  本质问题: 资源错配——专业能力没有部署在流量入口
  量化损失: "前端引流的钱，大部分都打了水漂"
```

#### 模型3：堵路三板斧

```yaml
原理: 在推你的方案之前，先让用户自己否定其他方案

三板斧:
  1. 预判: "你们肯定也搞过XX吧？"
  2. 追问: "为啥没解决？是A原因，还是B原因？"
  3. 让用户自己说: 让用户亲口说出"XX方案不行"
  
效果:
  - 用户自己说出来的比你说更有说服力
  - 用户会觉得你懂行，见过很多类似情况
  - 为你的方案扫清障碍
```

#### 模型4：概念重塑

```yaml
原理: 用新概念重新包装你的方案，锚定新的价值认知

操作:
  旧概念: "AI客服系统"
    用户联想: 机器人、不智能、冷冰冰、要花钱买系统
    
  新概念: "克隆你的金牌顾问"
    用户联想: 复制最牛的人、解决分身乏术、无限扩展
    
概念重塑的原则:
  1. 用用户熟悉的概念（克隆、大厨、原料）
  2. 直接击中用户的痛点
  3. 让用户一听就懂价值在哪里
```

#### 模型5：门槛降低术

```yaml
原理: 让用户觉得"这事简单，我能做"

技巧:
  1. 角色重新定义: "你只需要提供原料"
  2. 格式不限: "语音、文字、文档都行"
  3. 正常化顾虑: "十个老板九个都这么说"
  4. 提供替代方案: "没资料也行，我们可以访谈"
  5. 技术托管: "你不用懂技术，只要最后验收"
  
核心: 把用户的心理负担降到最低
```

#### 模型6：焦点转移术

```yaml
原理: 把用户的注意力从"要不要做"转移到"具体怎么做"

示例:
  不说: "你要不要试试我们的方案？"
  而说: "你那个金牌顾问，配合度怎么样？"
  
效果:
  - 暗示用户已经决定做了
  - 讨论的是执行细节，不是是否要做
  - 推进到更具体的下一步
```

### 3.3 高情商表达技巧库

```yaml
高情商表达:

  行业认知展示:
    - "这行我熟"
    - "这个是XX行业通病了"
    - "你这个问题太典型了"
    
  正常化:
    - "十个老板九个都这么说"
    - "这太正常了"

    - "有什么给什么，别有压力"
    - "这步很简单"
    - "你不用懂任何技术"
    
  坦诚表达:
    - "先解决80%的问题，后面再迭代到95分"
    
  控制权让渡:
    - "你觉得转化率能不上去吗？"
    - "你觉得怎么样？"
    - "只要她愿意配合，这事就好办"
```

## 五、核心洞察总结

### 这套系统的本质是什么？

```
不是"说服用户买东西"
而是"帮用户理清思路，让他自己得出结论"

具体方法:
1. 让用户自己说出痛点（二选一引导）
2. 让用户自己否定替代方案（堵路三板斧）
3. 让用户自己算清账（问题重新定义+量化损失）
4. 让用户自己得出"需要专业帮助"的结论（SPIN的N）
5. 让用户觉得"这事我能做"（门槛降低术）
6. 让用户讨论"怎么做"而不是"要不要做"（焦点转移）

整个过程中，用户感觉是自己在思考、在选择、在决定。
实际上，每一步都在你设计的框架里。
```

### 一句话总结

> **顾问式销售的本质是"控制权幻觉"——你在引导，但用户感觉是自己在选择。**
>
> 通过二选一框架、问题重新定义、堵路预判、概念重塑、门槛降低、焦点转移，让用户一步步走向你希望的结论，同时觉得"这是我自己想明白的"。
